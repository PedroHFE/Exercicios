Hierarquia de tabelas simples

Em primeiro lugar temos a tag table = tabela dentro de table temos, table row = linha de tabela, dentro do table row temos header = cabeçalho de tabela e table data = dado de tabela. O w3c diz que o fechamento das tags <td>(Table data) e <tr> (Table row) é opcional, vale lembra que se a hierarquia não for obedecida os dados vão aparecer antes ou depois da tabela.

A1 A2 A3
A2 B2 C2
A3 B3 C3

<table>
<tr>
<td>A1</td>
<td>A2</td>
<td>A3</td>
</tr>
<tr>
<td>A2</td>
<td>B2</td>
<td>C2</td>
</tr>
<tr>
<td>A3</td>
<td>B3</td>
<td>C3</td>
</tr>
</table>

Alinhamento da tabela

Para alinhar na horizontal, podemos usar a tag text-aling (left-center-right) e para alinhar na vertical, usamos a tag vertical-align(top-middle-bottom).

Coloque sempre o scope(escopo) das th(titulos), scope="col" para os dados que ficarão na coluna (a baixo), e scope="row" para os dados na mesma linha.

Tags semanticas para tabelas grandes

Nas tabelas grandes ainda usamos as tags tr(table row) e td(table data) mas agora aprenderemos a anatomia das tabelas grandes. Sempre vai começar com a tag table mas dentro de table, usaremos a Thead(cabeça da tabela), Tbody(corpo da tabela) e o Tfoot(pé da tabela) da tabela, para finalizar também podemos colocar uma legenda na tabela com o Caption, o caption você só coloca texto, nada de tr ou td.

Efeito zebrado

O efeito zebrado é muito recomendado quando a tabela é grande, podemos fazer esse efeito bem rapido e fácil no CSS. Usamos a pseudo classe :nth-child(), entre parênteses podemos usar 3(três) valores, são odd(impar), even(par) ou (n°)n. O n a cada x(numero) ele vai aplicar o efeito.
