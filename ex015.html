Criando formulários

Para toda criação de formulários, devem começar com a tag form, na sequência vai depender de como você planejou montar o seu formulário. Na criação do seu formulário você irá utilizar a tag input, e de acordo com a sua nescessidade vai mudar o tipo do seu input. Por exemplo, se precisar de nomes usará o input:text.

Todo input precisa de um nome e id, o nome é mais importante para o PHP, já o id é mais importante para o JavaScript e para o uso da tag label. Não se faz formulário sem relacionar o label ao seu input(ou seu elemento).

Métodos de envio de formulários

Existem dois métodos que são simples para envio de formulário. São eles: method="GET" e method="POST" o primeiro é o GET (padrão), é o método que coloca os dados do formulário na URL.

O método POST não deixa aparecer os dados digitados na URL. Lembre-se: os dados não estão 100% protegidos mesmo com o método POST, pois eles ainda podem aparecer no 'inspecionar > network > depois de digitar os dados no formulário e enviar > procure dentro de network o cadastro que você fez > payload > você vai achar os dados lá.

o POST nao protegeu os dados, ele apenas não mostrou os dados na URL. ainda dá pra capturar os dados se olhar dentro do corpo dos cabeçalhos das requisições da emissão dos dados. Pra proteger esses dados de fato, você precisa criptografar, e pra isso se utiliza HTTPS (outro protocolo de transmissão de dados). Pois o HTTP é o q estamos utilizamos.

Quando usar GET ou POST?

GET: basicamente, você vai utilizar quando os dados não foram sensíveis (ex: nome, peso, busca estilo Google) você NÃO estará pedindo senha, cartão de credito, endereço, CEP, envio de fotos ou qualquer tipo de arquivo. Utilize GET quando os dados NÃO forem sensíveis e não precisar serem protegidos. OBS: tem um limite, quando você envia dados por GET, só pode enviar até 3 mil bytes (é aproximadamente 3 mil letras).

POST: dados sensíveis (o ideal é com HTTPS para proteger dados), os dados não aparecem na URL, mais de 3 mil bytes, envio de arquivos.

Alguns tipos de controle de input

  • require: Requerimento, não permite enviar os dados vazios.
  • minlength: Mínimo de caracteres.
  • maxlength: Máximo de caracteres.
  • placeholder: Dica, pode colocar um texto indicativo.
  • autocomplete: Primeiro tem que colocar na tag form, "on" ou "off".
  • autocomplete: Pode colocar nos inputs indicando se é usuário, senha...
  • min: No input númerico, mínimo aceitavel.
  • max: No input númerico, máximo aceitavel.
  • step: No input númerico, para valores quebrados como 0.5, 1,3 e etc.
  • value: Já aperece sugerido dentro do input.
  • fieldset: Grupo de cantos(grupo de dados).
  • legend: Legenda.
  • range: Cria uma barra com níveis, que o usuário pode arrastar.
  • color: Cria uma caixa que permite o usuário escolha uma cor.
  • file: Para o usuário escolher e enviar arquivos.
  • checkbox: Cria uma caixinha para marcar e desmarcar.
  • radio: Cria uma bolinha para marcar.
  • pattern: Padrão, alguns inputs tem suas expressões regulares diferentes.

Por padrão, o range começa sempre em 50 (no meio), variando de 0 a 100, mas pode ser personalizado, utilizando os atributos value=" ", min=" " e max=" ", respectivamente.

Em input color você pode definir a cor inical usando o atributo value=" ". Como também definir algumas o número de outras cores mas usando outros inputs juntos.

Ao trabalhar com input:file, usar sempre o atributo method="POST" dentro da tag form, para que não haja uma limitação de 3.000 bites no envio do arquivo.

No input radio se tiver mais de uma opção não esqueça de colocar o name igual e o id diferentes, caso o contrário não conseguira desmarcar. No radio só pode ter um marcado diferente do checkbox.

Pattern configuramos uma expressão regular para definir qual será o formato aceito para o valor. Toda expressão regular começa com um '^' e termina com '$', dígitos para telefone pode inciar assim '\d'.

  • \([0-9] {2}\) vai indicar que é necessário colocar dois dígitos de 0 a 9 entre parênteses.
  • ([\s]) indica que logo em seguida, teremos um espaço, logo após o fechamento dos parênteses.
  • [0-9] {4,6}- configura que podem existir entre 4 e 6 dígitos numéricos entre 0 e 9 e logo depois, um traço.
  • [0-9] {4} por fim, o número se encerra com 4 dígitos entre 0 e 9.

Elemento Textarea

A tag textarea cria uma área de texto que vai aceitar múltiplas linhas. Nesse caso, os parâmetros rows(linhas) e cols(colunas) definem respectivamente em quantas linhaas e quantas colunas o conteúdo será exibido, mas saiba que o conteúdo completo pode ter muito mais que o número de linhas configurado. A capacidade de um <textarea> é ilimitada por padrão, você pode colocar um maxlength para limitar a quantidade de caracteres que o usuário pode digitar. Podemos usar o atributo style="resize: none" para evitar que o usuário altere o tamanho do textarea.

Elemento Select:

A tag de formulário select tem uma função muito parecida com a de um input.. Isso porque ela renderiza na tela um componente de entrada para o formulário. O elemento HTML select representa um controle que exibe um menu de opções. É um elemento que ao clicarmos, desce uma lista com diferentes opções para escolher.

No elemento select, o usuário não insere um dado digitando, ele simplesmente seleciona a opção que lhe atende... Portanto, use esse elemento para a entrada de dados que são imutáveis. Por exemplo: Usar o elemento select para escolher um dos estados do Brasil. Não existe necessidade de alguém eventualmente precisar escolher outro item que já não esteja na lista porque a composição dos estados é teoricamente imutável.
Nota: No elemento select, os atributos "name" e "id" também devem ser configurados normalmente... Ele também se enquadra em um componente de entrada como qualquer outro, portanto também precisa de uma label. Sendo que o VÍNCULO entre o componente select e sua respectiva label também deve ser estabelecido.

Depois de inserir o select no seu form, é preciso definir as opções que irão compor a lista de seleção dele. Para definir as opões do elemento, usamos a tag option. Essa tag deve ser inserida dentro do elemento select (option é sempre introduzido como filho de select nesse caso), sendo que ela tem a função de definir as opções aparecerão dentro do menu de seleção. Para cada opção da lista, um novo option deve ser aberto dentro de select.

O rótulo da opção (texto que representa o item de lista quando a caixa de seleção é aberta) deve ser inserido dentro do par de tags do option, por exemplo: <option>Santa Catarina</option>. Sendo que por padrão, quando uma tag option é aberta, um atributo "value" já acompanha a tag. É o conteúdo dentro de "value" que de fato será enviado para o servidor quando a submissão do form é executada. Seguindo o exemplo anterior, a nossa opção com o value configurado pode ser assim: <option value="SC">Santa Catarina</option>

Dentro de uma lista de seleção do elemento select, os options (itens da lista) podem ser AGRUPADOS em categorias intituladas, isso facilita a navegação do usuário. Por exemplo, na nossa lista com os estados do Brasil, poderíamos organizar esses estados pelas regiões do país, categorizar eles por região... Dessa maneira, quando a lista de seleção for aberta, os estados iriam aparecer agrupados por região, sendo que cada um desses grupos de estado teriam um título (sudeste, sul, centro-oeste e por aí vai).

Para fazer isso, utilizamos a tag optgroup. Para formar um grupo dentro da lista de seleção com o optgroup, você deve envelopar com essa tag os itens (options) que supostamente fazem parte daquele grupo. Sendo que obrigatoriamente, quando uma tag optgroup é aberta, temos que usar nela o atributo "label" (é um atributo que se chama label, não é uma tag label). O texto inserido dentro do atributo label que será exibido para o usuário como o título daquele grupo quando a lista for aberta.
Nota: Os itens da lista de seleção (options) ficam dispostos para o usuário na ordem de inserção do código HTML.

Atributo do select:

selected: O atributo selected serve para definirmos uma opção inicial padrão, ele deve ser inserido dentro da tag de abertura do option que você deseja definir como padrão. Dessa forma, quando a página com o elemento select for carregada, essa opção escolhida já vai aparecer por padrão selecionada no componente de entrada select, ficando a critério do usuário alterar a opção ou manter a padrão determinada por você.

Elemento Datalist

O elemento datalist funciona junto com o input:text uma caixa de texto comum. A única diferença é a presença dp atributo list(lista), que vai apontar para uma <datalist> que está semdo criada logo abaixo.

Assim como <select>, uma <datalist> também pode conter vários <option> em seu interior. Visualmente, a caixa de texto vai aceitar qualquer valor personalizado, mas sempre vai sugerir os valores que definirmos dentro das opções, inclusive filtrando os valores exibidos conforme vamos digitando o conteúdo da caixa de texto.

Elemento Output em formulários HTML

Um recurso que foi adicinado à HTML5 o elemento <output>, que pode ser usado para executar tarefas simples e gerar uma saída na tela. Essa execução é feita usando a Linguagem de Programação JavaScript e pode ser executada diretamente no atributo oniput do formulário.

A única novidade é onde temos um campo de saída que não pode ser editado pelo seu visitante e sequer será enviado pelo seu formulário. O seu objetivo aqui é manipular o seu conteúdo através de instruções simples. Atenção as saídas não são enviadas! O valor gerado dentro de um elemento <output> não será enviado quando pressionarmos o botão de submissão. Ele serve apenas para mostrar saídas ao seu visitante.
