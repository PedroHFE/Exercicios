Criando formulários

Para toda criação de formulários, devem começar com a tag form, na sequência vai depender de como você planejou montar o seu formulário. Na criação do seu formulário você irá utilizar a tag input, e de acordo com a sua nescessidade vai mudar o tipo do seu input. Por exemplo, se precisar de nomes usará o input:text.

Todo input precisa de um nome e id, o nome é mais importante para o PHP, já o id é mais importante para o JavaScript e para o uso da tag label. Não se faz formulário sem relacionar o label ao seu input(ou seu elemento).

Métodos de envio de formulários

Existem dois métodos que são simples para envio de formulário. São eles: method="GET" e method="POST" o primeiro é o GET (padrão), é o método que coloca os dados do formulário na URL.

O método POST não deixa aparecer os dados digitados na URL. Lembre-se: os dados não estão 100% protegidos mesmo com o método POST, pois eles ainda podem aparecer no 'inspecionar > network > depois de digitar os dados no formulário e enviar > procure dentro de network o cadastro que você fez > payload > você vai achar os dados lá.

o POST nao protegeu os dados, ele apenas não mostrou os dados na URL. ainda dá pra capturar os dados se olhar dentro do corpo dos cabeçalhos das requisições da emissão dos dados. Pra proteger esses dados de fato, você precisa criptografar, e pra isso se utiliza HTTPS (outro protocolo de transmissão de dados). Pois o HTTP é o q estamos utilizamos.

Quando usar GET ou POST?

GET: basicamente, você vai utilizar quando os dados não foram sensíveis (ex: nome, peso, busca estilo Google) você NÃO estará pedindo senha, cartão de credito, endereço, CEP, envio de fotos ou qualquer tipo de arquivo. Utilize GET quando os dados NÃO forem sensíveis e não precisar serem protegidos. OBS: tem um limite, quando você envia dados por GET, só pode enviar até 3 mil bytes (é aproximadamente 3 mil letras).

POST: dados sensíveis (o ideal é com HTTPS para proteger dados), os dados não aparecem na URL, mais de 3 mil bytes, envio de arquivos.

Alguns tipos de controle

  • require: Requerimento, não permite enviar os dados vazios.
  • minlength: Mínimo de caracteres.
  • maxlength: Máximo de caracteres.
  • placeholder: Dica, pode colocar um texto indicativo.
  • autocomplete: Primeiro tem que colocar na tag form, "on" ou "off".
  • autocomplete: Pode colocar nos inputs indicando se é usuário, senha...
  • min: No input númerico, mínimo aceitavel.
  • max: No input númerico, máximo aceitavel.
  • step: No input númerico, para valores quebrados como 0.5, 1,3 e etc.
  • value: Já aperece sugerido dentro do input.
  • fieldset: Grupo de cantos(grupo de dados).
  • legend: Legenda.
  • checkbox: Cria uma caixinha para marcar e desmarcar.
  • radio: Cria uma bolinha para marcar.
  • pattern: Padrão, alguns inputs tem suas expressões regulares diferentes.

No input radio se tiver mais de uma opção não esqueça de colocar o name igual e o id diferentes, caso o contrário não conseguira desmarcar. No radio só pode ter um marcado diferente do checkbox.

Pattern configuramos uma expressão regular para definir qual será o formato aceito para o valor. Toda expressão regular começa com um '^' e termina com '$', dígitos para telefone pode inciar assim '\d'.

  • \([0-9] {2}\) vai indicar que é necessário colocar dois dígitos de 0 a 9 entre parênteses.
  • ([\s]) indica que logo em seguida, teremos um espaço, logo após o fechamento dos parênteses.
  • [0-9] {4,6}- configura que podem existir entre 4 e 6 dígitos numéricos entre 0 e 9 e logo depois, um traço.
  • [0-9] {4} por fim, o número se encerra com 4 dígitos entre 0 e 9.
